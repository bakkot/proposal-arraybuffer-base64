    Stage 2 Draft / May 24, 2023

    Uint8Array to/from base64

    Contributing to this Proposal

    This proposal is developed on GitHub. A playground is available with examples and a non-production polyfill to allow direct experimentation.

    1 Uint8Array.prototype.toBase64 ( [ options ] )

    1. Let O be the this value.
    2. Let toEncode be ? GetUint8ArrayBytes(O).
    3. Let opts be ? GetOptionsObject(options).
    4. Let alphabet be ? Get(opts, "alphabet").
    5. If alphabet is undefined, set alphabet to "base64".
    6. Set alphabet to ? ToString(alphabet).
    7. If alphabet is neither "base64" nor "base64url", throw a TypeError exception.
    8. If alphabet is "base64", then
      1. Let outAscii be the sequence of code points which results from encoding toEncode according to the base64 encoding specified in section 4 of RFC 4648.
    9. Else,
      1. Assert: alphabet is "base64url".
      2. Let outAscii be the sequence of code points which results from encoding toEncode according to the base64url encoding specified in section 5 of RFC 4648.
    10. Return CodePointsToString(outAscii).
    11. NOTE: CodePointsToString is used only because RFC 4648 does not produce a sequence of code units. Implementations may be able to produce an ECMAScript string directly.

    2 Uint8Array.prototype.toPartialBase64 ( [ options ] )

    1. Let O be the this value.
    2. Let toEncode be ? GetUint8ArrayBytes(O).
    3. Let opts be ? GetOptionsObject(options).
    4. Let alphabet be ? Get(opts, "alphabet").
    5. If alphabet is undefined, set alphabet to "base64".
    6. Set alphabet to ? ToString(alphabet).
    7. If alphabet is neither "base64" nor "base64url", throw a TypeError exception.
    8. Let more be ToBoolean(? Get(opts, "more")).
    9. Let extra be ? Get(opts, "extra").
    10. If extra is neither undefined nor null, then
      1. TODO: consider handling array-of-bytes.
      2. Let extraBytes be ? GetUint8ArrayBytes(extra).
      3. Set toEncode to the list-concatenation of extraBytes and toEncode.
    11. If more is true, then
      1. Let toEncodeLength be the length of toEncode.
      2. Let overflowLength be toEncodeLength modulo 3.
      3. Let overflowBytes be a List whose elements are the last overflowLength elements of toEncode.
      4. TODO: convert overflowBytes to a new Uint8Array here.
      5. Remove the last overflowLength elements of toEncode.
    12. Else,
      1. Let overflowBytes be null.
    13. If alphabet is "base64", then
      1. Let outAscii be the sequence of code points which results from encoding toEncode according to the base64 encoding specified in section 4 of RFC 4648.
    14. Else,
      1. Assert: alphabet is "base64url".
      2. Let outAscii be the sequence of code points which results from encoding toEncode according to the base64url encoding specified in section 5 of RFC 4648.
    15. Let result be CodePointsToString(outAscii).
    16. NOTE: CodePointsToString is used only because RFC 4648 does not produce a sequence of code units. Implementations may be able to produce an ECMAScript string directly.
    17. Let obj be OrdinaryObjectCreate(%Object.prototype%).
    18. Perform ! CreateDataPropertyOrThrow(obj, "result", result).
    19. Perform ! CreateDataPropertyOrThrow(obj, "extra", overflowBytes).
    20. Return obj.

    3 Uint8Array.prototype.toHex ( )

    1. Let O be the this value.
    2. Let toEncode be ? GetUint8ArrayBytes(O).
    3. Let out be the empty String.
    4. For each byte byte of toEncode, do
      1. Let str be Number::toString(byte, 16).
      2. If the length of str is 1, set str to the string-concatenation of "0" and str.
      3. Set out to the string-concatenation of out and str.
    5. Return out.

    4 Uint8Array.fromBase64 ( value [ , options ] )

    1. Let string be ? GetStringForBinaryEncoding(value).
    2. Let opts be ? GetOptionsObject(options).
    3. Let alphabet be ? Get(opts, "alphabet").
    4. If alphabet is undefined, set alphabet to "base64".
    5. Set alphabet to ? ToString(alphabet).
    6. If alphabet is neither "base64" nor "base64url", throw a TypeError exception.
    7. TODO: normalize whitespace / padding here.
    8. TODO: figure out what the right defaults for whitespace/padding are.
    9. Let characters be StringToCodePoints(string).
    10. NOTE: StringToCodePoints is used only because RFC 4648 does not produce a sequence of code units. Implementations may be able to base64-decode string directly.
    11. If alphabet is "base64", then
      1. If characters cannot result from applying the base64 encoding specified in section 4 of RFC 4648 to some sequence of bytes, throw a SyntaxError exception.
      2. Let bytes be the unique sequence of bytes such that applying the base64 encoding specified in section 4 of RFC 4648 to that sequence would produce characters.
    12. Else,
      1. Assert: alphabet is "base64url".
      2. If characters cannot result from applying the base64url encoding specified in section 5 of RFC 4648 to some sequence of bytes, throw a SyntaxError exception.
      3. Let bytes be the unique sequence of bytes such that applying the base64url encoding specified in section 5 of RFC 4648 to that sequence would produce characters.
    13. Let resultLength be the number of bytes in bytes.
    14. Let result be ? AllocateTypedArray("Uint8Array", %Uint8Array%, %Uint8Array.prototype%, resultLength).
    15. Set the value at each index of result.[[ViewedArrayBuffer]].[[ArrayBufferData]] to the value at the corresponding index of bytes.
    16. Return result.

    5 Uint8Array.fromPartialBase64 ( value [ , options ] )

    1. Let string be ? GetStringForBinaryEncoding(value).
    2. Let opts be ? GetOptionsObject(options).
    3. Let alphabet be ? Get(opts, "alphabet").
    4. If alphabet is undefined, set alphabet to "base64".
    5. Set alphabet to ? ToString(alphabet).
    6. If alphabet is neither "base64" nor "base64url", throw a TypeError exception.
    7. Let more be ToBoolean(? Get(opts, "more")).
    8. Let extra be ? Get(opts, "extra").
    9. If extra is neither undefined nor null, then
      1. Let extraString be ? GetStringForBinaryEncoding(extra).
      2. Set string to the list-concatenation of extraString and string.
    10. If more is true, then
      1. TODO: think about handling of padding on string / extra in this case. This currently assumes no padding on either.
      2. Let stringLength be the length of string.
      3. Let overflowLength be stringLength modulo 4.
      4. Let newLength be stringLength - overflowLength.
      5. Let overflow be the substring of string from newLength.
      6. Set string to the substring of string from 0 to newLength.
    11. Else,
      1. Let overflow be null.
    12. TODO: normalize whitespace / strip padding here.
    13. TODO: figure out what the right defaults for whitespace/padding are.
    14. Let characters be StringToCodePoints(string).
    15. NOTE: StringToCodePoints is used only because RFC 4648 does not produce a sequence of code units. Implementations may be able to base64-decode string directly.
    16. If alphabet is "base64", then
      1. If characters cannot result from applying the base64 encoding specified in section 4 of RFC 4648 to some sequence of bytes, throw a SyntaxError exception.
      2. Let bytes be the unique sequence of bytes such that applying the base64 encoding specified in section 4 of RFC 4648 to that sequence would produce characters.
    17. Else,
      1. Assert: alphabet is "base64url".
      2. If characters cannot result from applying the base64url encoding specified in section 5 of RFC 4648 to some sequence of bytes, throw a SyntaxError exception.
      3. Let bytes be the unique sequence of bytes such that applying the base64url encoding specified in section 5 of RFC 4648 to that sequence would produce characters.
    18. Let resultLength be the number of bytes in bytes.
    19. Let result be ? AllocateTypedArray("Uint8Array", %Uint8Array%, %Uint8Array.prototype%, resultLength).
    20. Set the value at each index of result.[[ViewedArrayBuffer]].[[ArrayBufferData]] to the value at the corresponding index of bytes.
    21. Let obj be OrdinaryObjectCreate(%Object.prototype%).
    22. Perform ! CreateDataPropertyOrThrow(obj, "result", result).
    23. Perform ! CreateDataPropertyOrThrow(obj, "extra", overflow).
    24. Return obj.

    6 Uint8Array.fromHex ( value )

    1. Let string be ? GetStringForBinaryEncoding(value).
    2. TODO: consider stripping whitespace here.
    3. Let stringLen be the length of string.
    4. If stringLen modulo 2 is not 0, throw a SyntaxError exception.
    5. If string contains any code units which are not in "0123456789abcdefABCDEF", throw a SyntaxError exception.
    6. Let resultLength be stringLen / 2.
    7. Let result be ? AllocateTypedArray("Uint8Array", %Uint8Array%, %Uint8Array.prototype%, resultLength).
    8. Let index be 0.
    9. Repeat, while index < resultLength,
      1. Let stringIndex be index * 2.
      2. Let hexits be the substring of string from stringIndex to stringIndex + 2.
      3. Let byte be the integer value represented by hexits in base-16 notation, using the letters A-F and a-f for digits with values 10 through 15.
      4. Perform ! IntegerIndexedElementSet(result, 𝔽(index), 𝔽(byte)).
      5. Set index to index + 1.
    10. Return result.

    7 GetUint8ArrayBytes ( ta )

    The abstract operation GetUint8ArrayBytes takes argument ta (an ECMAScript language value) and returns either a normal completion containing a List of byte values or a throw completion. It performs the following steps when called:

    1. Perform ? RequireInternalSlot(ta, [[TypedArrayName]]).
    2. If ta.[[TypedArrayName]] is not "Uint8Array", throw a TypeError exception.
    3. Let buffer be ta.[[ViewedArrayBuffer]].
    4. If IsDetachedBuffer(buffer) is true, throw a TypeError exception.
    5. Let block be buffer.[[ArrayBufferData]].
    6. Let offset be ta.[[ByteOffset]].
    7. Let length be ta.[[ArrayLength]].
    8. Return a List whose elements are the byte values in block starting at offset and continuing for a total of length bytes, in order.

    8 GetStringForBinaryEncoding ( arg )

    The abstract operation GetStringForBinaryEncoding takes argument arg (an ECMAScript language value) and returns either a normal completion containing a String or a throw completion. It performs the following steps when called:

    1. If arg is an Object, let string be ? ToPrimitive(arg, string); else let string be arg.
    2. NOTE: Because [ is not a valid base64 or hex character, the Strings returned by %Object.prototype.toString% will produce a SyntaxError during encoding. Implementations are encouraged to provide an informative error message in that situations.
    3. if string is not a String, throw a TypeError exception.
    4. NOTE: The above step is included to prevent errors such as accidentally passing null to fromBase64 and receiving a Uint8Array containing the bytes « 0x9e, 0xe9, 0x65 ».
    5. Return string.

    9 GetOptionsObject ( options )

    1. If options is undefined, then
      1. Return OrdinaryObjectCreate(null).
    2. If Type(options) is Object, then
      1. Return options.
    3. Throw a TypeError exception.
